When to backup:

• Before any bulk upload
• Before every deployment
Restore from Backup
This overwrites the live database. All current data will be replaced with the uploaded file. The current DB is auto-saved as an emergency copy before overwrite.
Only SQLite .db files accepted. File is validated before restore.
Restore Procedure
  1. Go to your Google Drive / SharePoint → TMS Backups folder
  2. Download the most recent training_YYYY-MM-DD.db file
  3. Come back here → upload that file → click Restore Database
  4. All data restored immediately — users can log in normally
